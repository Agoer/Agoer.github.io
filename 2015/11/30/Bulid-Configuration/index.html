Build Configuration

Xcode默认会有2个编译模式，一个是Debug，一个是Release。Release下不能调试程序，编译时有做编译优化，会比用Debug打包出来的运行快，另外包也会更小。

为什么会有这样的编译模式？如果我有三个不同的API地址，分别对应 开发测试服务器（ test_debug ）、测试服务器( debug )、预发布线上服务器（Beta）、线上服务器( release ).如何能够不去修改代码的情况下去快速切换调试模式呢 ？
幸好有 bulid configuration！

配置不同的 APP Name

1.Target - Build Settings里点+号，添加一个User-Defined Setting

2.比如我们请一个叫APP_DISPLAY_NAME的key值，下面BETA叫BETA，DEVELOP叫DEVELOP， Release还是用系统原配置。

3.进到Target - Info里，修改Bundle name为$(APP_DISPLAY_NAME)

效果如下：

  • debug //开发环境

  • beta //预发布线上环境(企业版)

  • release //线上环境
